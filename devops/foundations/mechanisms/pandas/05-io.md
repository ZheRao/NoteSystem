## Core Pattern

### 1. Read workbook metadata

```python
import pandas as pd

xls = pd.ExcelFile("inputs.xlsx")

print(xls.sheet_names)
```

Example output:

```python
[
    "README",
    "Crop_Inputs",
    "Cash_Flow_Assumptions"
]
```

This is lightweight and does not load all sheets yet.


### 2. Validate required sheets

```python
EXPECTED_SHEETS = {
    "Crop_Inputs",
    "Cash_Flow_Assumptions"
}

missing = EXPECTED_SHEETS - set(xls.sheet_names)

if missing:
    raise ValueError(f"Missing required sheets: {missing}")
```

This is defensive engineering.

You fail early instead of silently producing corrupted downstream outputs.


### 3. Read a sheet

```python
df_crop = pd.read_excel(
    "inputs.xlsx",
    sheet_name="Crop_Inputs",
    usecols=REQUIRED_COLUMNS,
    dtype={
        "client_id": "string",
        "scenario_id": "string",
        "crop_type": "string",
        "acres": "float64"
    }
)

print(df_crop.head())
```

or using the already-open workbook:

```python
df_crop = pd.read_excel(
    xls,
    sheet_name="Crop_Inputs"
)
```


### 4. Validate columns

This is the critical systems step.

```python
REQUIRED_COLUMNS = {
    "Crop_Inputs": [
        "client_id",
        "scenario_id",
        "crop_type",
        "acres",
        "yield_per_acre"
    ]
}
```

Validation:

```python
required = set(REQUIRED_COLUMNS["Crop_Inputs"])
actual = set(df_crop.columns)

missing_cols = required - actual

if missing_cols:
    raise ValueError(
        f"Crop_Inputs missing columns: {missing_cols}"
    )
```


### 5. Normalize column names (HIGHLY recommended)

Excel users WILL produce:

```text
Crop Type
crop_type
CropType
crop type
 crop_type
```

Normalize immediately:

```python
df_crop.columns = (
    df_crop.columns
    .str.strip()
    .str.lower()
    .str.replace(" ", "_")
)
```

This single step prevents massive pain later.


### 6. Read all sheets at once

Useful later:

```python
all_sheets = pd.read_excel(
    "inputs.xlsx",
    sheet_name=None
)
```

Result:

```python
{
    "Crop_Inputs": DataFrame,
    "Cash_Flow_Assumptions": DataFrame,
}
```

## Important Excel Realities

You are about to discover something important:

Excel files are NOT clean databases.

Expect:

* merged cells
* hidden rows
* formulas
* blank rows
* weird typing
* duplicate columns
* inconsistent tabs
* manually inserted notes
* accidental formatting damage

So your ingestion layer should become:

```text
strict
bounded
defensive
auditable
```

Never trust Excel structure blindly.


## One Important Decision

You should decide EARLY:

### Option A — Strict Template Workbook

Users must use YOUR exact template.

Pros:

* reliable
* easy validation
* scalable

Cons:

* less flexible


### Option B — Flexible Workbook Parsing

You try to infer structure.

Pros:

* user friendly

Cons:

* complexity explosion
* fragile logic
* high maintenance


For your current trajectory:

> Lean heavily toward strict templates.

That aligns with your defensive engineering evolution.
